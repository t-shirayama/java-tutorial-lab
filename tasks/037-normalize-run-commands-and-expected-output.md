# タスク: 全章の実行コマンドと期待出力の形式を統一する

## 背景

読者が手元で確認しやすいように、各章の実行方法と期待出力の形式を揃えます。

## 対象

- `docs/*/README.md`

## 修正内容

- 各章にDocker実行コマンドを同じ形式で記載する。
- ローカルMaven実行コマンドを同じ形式で記載する。
- 可能な章には `期待される出力例` を追加する。
- 既存の `docker compose exec -w /workspace/docs/<chapter>/examples java mvn compile exec:java` 方針と矛盾しないようにする。

## 完了条件

- 全章の実行手順が同じ読み味になっている。
- 期待出力を見れば、読者が成功したか判断できる。
- `_site`生成後もコマンドが読みやすい。
