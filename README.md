# Javaチュートリアル

Javaを基礎から順番に学ぶためのハンズオン形式のリポジトリです。

各章では、まず説明を読み、次にサンプルコードをMavenで実行し、最後に自分でコードを書き換えて動きを確認します。読むだけで終わらせず、必ず手元で動かしながらJavaの考え方と書き方を身につけることを目指します。

## 対象読者

- Javaを基礎から体系的に学びたい人
- Javaの文法だけでなく、オブジェクト指向、コレクション、ラムダ式、ストリーム、例外、スレッドなどを手を動かして理解したい人
- Mavenを使ってJavaのサンプルコードを実行する流れに慣れたい人
- 将来的にSpringなどのJavaエコシステムへ進みたい人

## 前提環境

次のどちらかの方法で実行環境を用意してください。

- ローカルにJava 21 LTSとApache Mavenをインストールする
- DockerとDocker Composeを使う

JavaのバージョンはJava 21 LTSを基準にします。`record`、`sealed`、仮想スレッド、FFM APIなど、近年のJavaで重要になっている機能もJava 21を前提に扱います。

## Dockerで実行する方法

このリポジトリには、Java 21とMavenをまとめた学習用のDocker環境を用意しています。ローカルにJDKやMavenを直接入れなくても、各章のサンプルを実行できます。

初回セットアップ:

```bash
docker compose up -d --build
```

このコマンドは、Java学習用コンテナをビルドしてバックグラウンドで起動します。

日常的な実行:

```bash
docker compose exec -w /workspace/docs/01-overview/examples java mvn compile exec:java
```

章が増えても、`-w /workspace/docs/<chapter>/examples` の部分だけを変えれば同じ形で実行できます。

学習を終えてコンテナを止めたいとき:

```bash
docker compose down
```

`docker compose run` はコマンド実行のたびに新しいコンテナを起動します。`docker compose exec` は、すでに起動しているコンテナの中でコマンドを実行するため、繰り返し試しやすくなります。

## READMEをHTMLで読む方法

ルートREADMEと各章のREADMEは、GitHub ActionsでHTMLに変換し、GitHub Pagesで公開します。トップページと章ごとのページに分けて生成します。リポジトリ設定のPages Sourceは`GitHub Actions`を選んでください。

ローカルでHTMLを生成する場合:

```bash
python3 -m pip install -r requirements.txt
python3 scripts/build_readme_html.py
```

生成されたHTMLは`_site/index.html`と`_site/chapters/`以下に出力されます。コードブロックはPython-MarkdownとPygmentsでシンタックスハイライトします。

## 学習の進め方

1. 章フォルダの`README.md`を読む
2. 同じ章フォルダの`examples/`にあるサンプルコードをMavenで実行する
3. サンプルコードを少し変更して、結果がどう変わるか確認する
4. 各章の参考資料で公式ドキュメントを確認する

サンプルコードは、章ごとにMavenで実行できる形に揃えます。具体的な実行コマンドは各章の`README.md`に記載します。ローカル環境で実行しても、Docker経由で実行しても構いません。

## 学習ロードマップ

章数が多いため、最初から全章を一気に読む必要はありません。目的に合わせて、次の順番を目安にしてください。

| コース | 対象章 | 目的 |
| --- | --- | --- |
| 最短コース | 1〜6章、8章、12〜14章、24章 | Javaの実行、基本文法、クラス、コレクション、制御構文、例外、エラー対応を最短で押さえる |
| 実務基礎コース | 1〜14章、16章、18〜19章、24〜25章 | 小さな業務コードを読み書きするための基礎、数値、パッケージ、ジェネリック、テスト、ファイル、日付時刻、ログを学ぶ |
| モダンJavaコース | 7章、9〜10章、20〜23章 | record、sealed、ラムダ式、Stream、スレッド、同時実行、Web技術、FFM APIなど近年のJavaの特徴を学ぶ |
| 発展コース | 20〜23章 | 並行処理、外部データ連携、外部関数・外部メモリなど、必要になってから深掘りする |

23章のFFM APIは発展編です。通常のWebアプリケーションや業務アプリケーションから始める場合は、先に1〜19章、24〜25章を固めてから読むと理解しやすくなります。

## フォルダ構成

ドキュメントは1章ごとに1フォルダへ分割します。節ごとの説明は章フォルダ内の`README.md`に見出しとしてまとめ、サンプルコードは同じ章フォルダの`examples/`に置きます。

```text
docs/
  01-overview/
    README.md
    examples/
  02-java-programming-basics/
    README.md
    examples/
  03-strings/
    README.md
    examples/
  ...
  23-ffm-api/
    README.md
    examples/
  24-errors-ide-debugging/
    README.md
    examples/
  25-practical-basics/
    README.md
    examples/
  26-capstone-task-cli/
    README.md
    examples/
  27-glossary/
    README.md
```

フォルダ名は`01-english-slug`形式で統一します。日本語の章名や節名はREADME本文と見出しで表現し、パス名は扱いやすい英語にします。

## 参照元の方針

このリポジトリの説明は、できるだけ一次情報を根拠にします。

一次情報として優先するもの:

- Oracle Docs
- OpenJDK
- Spring Official
- Apache Maven

二次情報として補助的に利用するもの:

- Qiita
- Zenn

QiitaやZennの記事は、理解を助ける補助情報としてのみ使います。このリポジトリの説明に反映する場合は、公式ドキュメントなどの一次情報で事実確認できた内容だけを採用します。

各章の`README.md`末尾には`参考資料`セクションを設け、公式リンクと補助的な二次情報を分けて記載します。
